## 1. 系统概述

本系统基于 Arduino UNO R3 与 ORP（氧化还原电位）模组构建，用于实验室场景下的 **pH 自动滴定控制** 与 **VFA（挥发性脂肪酸）测量**。系统支持两种工作模式：

- **脱机独立模式**：按下 Arduino 上的 D4 按键可在空闲时直接启动联合 VFA/ALK 测量；该测量要求使用未经普通 pH 调节的原始样品，无需 PC 连接，OLED 屏实时显示状态。
- **PC 联机模式**：通过 Python 上位机实时监测 pH 曲线、手动/自动控制三路蠕动泵、记录数据并导出 CSV 文件，同时可进行多点标定。

### 系统组成

| 部件 | 说明 |
|------|------|
| Arduino UNO R3 | 主控制器，运行固件 `xy_orp_arduino.ino` |
| ORP 模组 | 测量 ORP 毫伏值，型号兼容 Modbus RTU 协议，地址 0xFE |
| OLED 显示屏 | I2C 接口，128x64 像素，地址 0x3C，显示实时状态 |
| D4 按键 | 启动/停止脱机自动流程，内置上拉（INPUT_PULLUP） |
| 碱泵继电器 | D13 引脚控制，滴加碱液（NaOH 等） |
| 酸泵继电器 | D12 引脚控制，滴加酸液（HCl 等） |
| 水泵继电器 | D11 引脚控制，滴加稀释水或清洗液 |
| PC 上位机 | Python tkinter + matplotlib GUI，实时监测与控制 |

### 2.1 接线总表

| Arduino 引脚 | 连接外设 | 信号说明 | 备注 |
|-------------|---------|---------|------|
| 5V | ORP 模组 VIN | 供电 | 模组工作电压 5V |
| GND | ORP 模组 GND | 共地 | |
| D2 | ORP 模组 TX | SoftwareSerial RX | Arduino 接收 ORP 数据 |
| D3 | ORP 模组 RX | SoftwareSerial TX | Arduino 发送指令到 ORP |
| A4 (SDA) | OLED 屏 SDA | I2C 数据线 | |
| A5 (SCL) | OLED 屏 SCL | I2C 时钟线 | |
| 5V | OLED 屏 VCC | 供电 | |
| GND | OLED 屏 GND | 共地 | |
| D4 | 按键 | 一端接 D4，一端接 GND | 内置上拉，按下为 LOW |
| D13 | 碱泵继电器 IN | HIGH=开泵, LOW=关泵 | 碱液蠕动泵 |
| D12 | 酸泵继电器 IN | HIGH=开泵, LOW=关泵 | 酸液蠕动泵 |
| D11 | 水泵继电器 IN | HIGH=开泵, LOW=关泵 | 水泵/清洗蠕动泵 |
| D0 (RX) | PC USB 转串口 TX | 硬件串口接收 | PC 发送指令到 Arduino |
| D1 (TX) | PC USB 转串口 RX | 硬件串口发送 | Arduino 上报数据到 PC |

### 2.2 接线注意事项

1. **ORP 模组** 采用 Modbus RTU 协议，波特率 9600 bps，地址 0xFE。TX->D2, RX->D3 必须反接（即交叉连接）。
2. **OLED 屏** I2C 地址 0x3C，上电后自动初始化，无需额外配置。
3. **继电器模组** 请使用低电平触发或高电平触发的继电器模组（固件以 HIGH 为开泵）。如使用低电平触发模组，需在代码中修改输出逻辑或加装反相器。
4. **按键** 为常开触点按键，按下时 D4 读为 LOW。固件内部已启用 INPUT_PULLUP，无需外接上拉电阻。
5. **蠕动泵** 务必在通电前确认泵管安装方向，确保液体从试剂瓶流向反应容器。

## 4. 首次安装与部署

### 步骤一：安装 Python 依赖

打开命令提示符（Win+R 输入 cmd）执行：

```bash
pip install pyserial matplotlib
```

### 步骤二：上传 Arduino 固件

1. 用 USB 线连接 Arduino UNO 到 PC。
2. 打开 Arduino IDE，加载固件文件 `例程/Arduino UNO例程/xy_orp_arduino/xy_orp_arduino.ino`。
3. 选择 "工具 -> 开发板 -> Arduino Uno"，选择正确的 COM 端口。
4. 点击"上传"按钮，等待编译和烧录完成。
5. 上传成功后，打开串口监视器（波特率 115200），应看到 `ORP PH V10 START` 字样。

### 步骤三：连接硬件

按 [2.1 接线总表](#21-接线总表) 完成所有接线。接线完成后给 Arduino 上电，OLED 屏幕应显示：

```
READY. PRESS BTN
V1:0.0 V2:0.0ML
B:10.0 A:10.0 W:10.0
TGT:5.0 TOL:0.20
```

### 步骤四：进行 ORP-pH 标定

在 PC 上位机中完成多点标定（详见 [第 10 章](#10-标定与历史管理)），获得准确的 K 和 B 系数后，系统即可投入使用。

## 5. PC 上位机使用指南

### 5.1 启动上位机

进入 `上位机` 目录，双击 `启动.bat`，或在命令行中执行：

```bash
cd 上位机
python orp_monitor.py
```

> 若提示缺少依赖，请先执行 `pip install pyserial matplotlib`。

### 5.3 连接 Arduino

1. 在 "COM口" 下拉框中选择 Arduino 对应的串口（如 COM3）。
2. 波特率保持默认的 115200。
3. 点击"连接"按钮，状态栏变为绿色 "已连接: COM3" 即表示连接成功。
4. 连接成功后，左侧数据面板开始实时更新，pH 曲线开始绘制。
5. 当前版本中，上位机左侧主显示 pH、曲线、记录、PC 自动滴定和 VFA 启动准入统一以 `ORPMV` 配合当前本地 K/B 换算结果为准；MCU 上报的 `PH:` 仅保留为诊断值，不再覆盖主显示。

> 提示：如果下拉框中没有看到 COM 口，点击"刷新"按钮。确保 Arduino 已通过 USB 连接并且其他程序（如 Arduino IDE 串口监视器）未占用该端口。

### 5.4 手动泵控制

在"手动控制"区域：

- 点击"开"按钮即可手动启动对应泵（碱泵/酸泵/水泵）。
- 点击"关"按钮停止对应泵。
- 点击"全部停止"可同时关闭三路泵。
- 点击"重置体积统计"可将三路泵的累积体积归零（同时发送 RSTVOL 指令到 Arduino 清除 EEPROM 体积数据）。

### 5.5 PC 端自动滴定

PC 端内置独立的滴定状态机，与 Arduino 脱机状态机互不干扰：

1. 勾选"启用自动滴定"。
2. 选择滴定方向：加碱（pH 偏低时）或加酸（pH 偏高时）。
3. 设置目标 pH、触发 pH 和允许误差。
   - 点击“普通pH调节-启动”或启用自动滴定前，上位机会先下发并等待 `TT/TP/TL/TM/TD/TK/TB` 全部 ACK；在 ACK 完成前不会真正 `START`。
4. 系统自动分级开泵：
   - 当前 pH 与目标差距 > 1.0 -> 开泵 5 秒
   - 差距 0.5 ~ 1.0 -> 开泵 2 秒
   - 差距 0.2 ~ 0.5 -> 开泵 1 秒
   - 差距 < 允许误差 -> 停止滴定，显示达标
5. 每次开泵后自动等待混合时间（默认 5 秒），然后重新检测。

> 注意：PC 端滴定和 Arduino 脱机流程共享同一套硬件（泵），不要同时启动两者。进行脱机流程前请取消"启用自动滴定"。

## 6. 脱机独立运行模式

脱机模式的联合 VFA/ALK 测量是系统核心功能之一：不连接 PC，仅通过 Arduino + OLED 屏 + D4 按键即可在空闲时直接启动 **原始样品 VFA/ALK 联合测量**。普通 pH 调节与联合 VFA/ALK 是两个独立入口，达到“普通调节目标 pH”不是 VFA 前置步骤。

### 6.1 启动流程

1. 确保系统供电正常，OLED 显示 `READY. PRESS BTN`。
2. **按下 D4 按键**（短按一次），系统在空闲时直接开始以下联合测量步骤：

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  RS_IDLE  │───>│ RS_VFA   │───>│酸到 pH5.1│───>│酸到 pH3.5│───>│ RS_DONE │
│  (待机)   │    │10秒观察  │    │ 与短暂停顿│    │ 计算结果 │    │ (完成)  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

### 6.2 各阶段详解

**阶段 1 -- 10秒观察起始 pH (RS_VFA)**

- 联合 VFA/ALK 必须直接使用原始样品，禁止先按普通 pH 调节目标把样品调到 5.0。
- 固件连续观察 10 秒 EstPH，记录平均值、最小值和最大值。
- 只有当 10 秒平均值 `>= 5.5` 且极差 `<= 0.10` 时才允许进入后续酸滴定。
- 若平均值 `< 5.5`，会返回 `VFA:REJECT LOW_PH ...`，本次无新结果，必须重新取符合条件的原始样品。
- 若极差 `> 0.10`，会返回 `VFA:REJECT UNSTABLE ...`，本次无新结果，必须等待样品稳定或重新取样。

**阶段 2 -- VFA/ALK 酸滴定**

- 自动执行 Anderson-Yang 两点滴定法。
- 以 10 秒窗口平均起始 pH 作为计算输入。
- 两个阶段都采用 VFA 专用自适应脉冲滴定：根据当前 pH 与本阶段目标的差距，决定本次酸泵脉冲剂量。
- 当前脉冲时长分级与普通滴定保持一致：差值 `>1.0` 开 5 秒，`>0.5` 开 2 秒，尚未达到本阶段终点开 1 秒。
- 每次脉冲后必须立即停泵、累计本次酸体积、进入混合等待，并在复测到新 pH 后才决定下一脉冲或判定终点。
- 第一阶段到达 pH 5.1 后，必须先完成混合和复测，确认后才进入第二阶段 pH 3.5。
- 固件使用 `mix_wait` 作为阶段间混合等待；正式滴定部分从 10 秒观察结束后开始单独计时，最长允许 `1000 秒`，观察阶段不计入这 1000 秒。
- 达到 `1000 秒` 硬截止时，系统会安全停泵、保存已累计体积、返回空闲并明确上报 `TIMEOUT`；不会再由较小的阶段预算或脉冲次数上限提前误杀正常慢滴定。
- 详细原理见 [第 7 章](#7-vfa-测量原理与流程)。

**阶段 3 -- 完成 (RS_DONE)**

- OLED 显示最终 pH 值、VFA 结果（mM）、碱度（mM）、酸用量等信息。
- 结果保持显示 15 秒后进入完成状态。
- 按下 D4 按键可复位到待机状态；在空闲时再次按 D4 会重新启动新的原始样品联合测量。

### 6.3 中断流程

- 在上位机连接的情况下，发送 `STOP` 指令可随时中断脱机流程。
- 流程被中断后，所有泵立即停止，系统恢复到待机状态。

## 7. VFA 测量原理与流程

### 7.0 启动前条件

- 使用新的、未经普通 pH 调节的原始样品。
- 启动前确认当前起始 pH/EstPH 应在 `>= 5.5` 且状态稳定。
- 关闭 PC 自动滴定，等待普通泵和混合过程全部结束。
- 直接点击“开始VFA/ALK”或在空闲时按 D4；不要先调到“普通调节目标 pH”。
- 若曾把同一份样品调到普通目标 pH 5.0，再启动联合 VFA/ALK 时即使固件正确拒绝，也不会产生新结果；必须重新取原始样品。

### 7.1 Anderson-Yang 两点滴定法

本系统采用 Anderson-Yang 方法测量 VFA（挥发性脂肪酸）和碱度（Alkalinity），该方法只需滴定到两个 pH 终点（pH 5.1 和 pH 3.5），比传统五点法更快且重复性好。

**化学原理**：

- 第一阶段（pH 初始 -> pH 5.1）：酸中和样品中的碳酸氢盐缓冲体系，此阶段酸用量用于计算碱度。
- 第二阶段（pH 5.1 -> pH 3.5）：酸进一步滴定至 VFA 的 pKa 范围，此阶段酸用量用于计算 VFA 浓度。
- 论文验证的原始样品起始 pH 范围约为 5.5–7.65；本系统据此在固件中执行 `10秒平均值 >= 5.5` 的准入判断。

**计算公式**：

系统使用碳酸的一级和二级解离常数（K1 = 6.6e-7, K2 = 2.4e-5），通过求解二元一次方程组计算出 VFA 和 HCO3 浓度：

```
VFA_total (mM) = VFA_dissociated * (H+ + K2) / K2
Alkalinity (mM) = HCO3 (mM)
```

其中 `VFA_dissociated` 和 `HCO3` 由 Appendix B 的二元一次方程联立求出：

```
V1 = U2 - U1
V2 = U3 - U1 - 0.25
X  = (C2*B1 - C1*B2) / (B1*A2 - B2*A1)
Y  = (C1*A2 - C2*A1) / (B1*A2 - B2*A1)
```

- `X` 为 dissociated VFA，随后再换算为 `VFA_total`。
- `Y` 为 bicarbonate，也就是 OLED 上紧凑显示的 `ALK`。
- 当前固件上报的 `A1` 表示第一阶段到 pH 5.1 的酸增量；`A2` 表示第二阶段从 pH 5.1 到 3.5 的酸增量；`TOTAL` 表示到 pH 3.5 的累计酸量。
- 计算中的 `V2` 使用 `TOTAL - 0.25 mL`。其中 `0.25 mL` 是当前兼容实现里的空白滴定修正值，不是固定设备误差；后续应允许独立配置或标定。

### 7.2 关键参数

| 参数 | 指令 | 默认值 | 说明 |
|------|------|--------|------|
| 酸浓度 | `FN` | 0.1 N | 滴定用酸的标准当量浓度 |
| 样品体积 | `FS` | 50.0 ml | 被测样品的体积 |
| 终点 pH1 | 内置 | 5.1 | 第一终点（碱度终点） |
| 终点 pH2 | 内置 | 3.5 | 第二终点（VFA 终点） |

### 7.3 结果解读

- **VFA**：挥发性脂肪酸总浓度，单位 mM（毫摩尔/升），是厌氧消化过程的关键监测指标。
- **ALK**：OLED 上用紧凑标签 `ALK` 表示碳酸氢根结果，单位 mM。
- 结果通过串口以 `VFA_RAW:12.5,ALK_RAW:45.3,VFA:13.1,ALK:43.8` 格式输出，同时显示在 OLED 和 PC 上位机上。

## 8. OLED 显示屏状态说明

OLED 屏为 128x64 像素 I2C 显示屏（地址 0x3C），固件采用裸 I2C 驱动，内嵌 5x7 像素西文字体，每屏可显示 20 列 x 8 行字符。

### 8.1 各状态显示内容

**待机状态 (IDLE)**

```
READY. PRESS BTN
V1:0.0 V2:0.0ML       ← V1=碱体积, V2=酸体积
B:10.0 A:10.0 W:10.0  ← B=碱流速, A=酸流速, W=水流速
TGT:5.0 TOL:0.20      ← 目标pH 与 允许误差
```

**滴定中 (TITRATING)**

```
TITRATING...
PH:4.52->5.00          ← 当前pH 与 目标pH
PUMP:BASE 5S           ← 当前泵(BASE=碱/ACID=酸) 与 剩余秒数
DOSE#3 GAP:0.48        ← 滴加次数 与 当前差距
```

**混合中 (MIXING)**

```
MIXING...   3S         ← 剩余混合秒数
PH:4.52                ← 当前pH
V1:1.5 V2:3.2ML        ← 累积碱/酸体积
TGT:5.0 TOL:0.20       ← 目标与误差
```

**VFA 测量中 (VFA MEASURING)**

```
VFA MEASURE
PH0:6.89 PH:5.10       ← 固定起始平均 pH / 当前实时 pH
S1 DOSING              ← 当前为第 1 阶段滴定或混合
V1:2.3                 ← Stage1 显示当前 V1 增量
```

Stage2 过程中底行会改为：

```
V1:3.4 V2:1.8          ← 固定已完成 V1 / 当前 V2 增量
```

Stage1/Stage2 已统一为“脉冲滴定 + 混合复测”闭环显示，不再存在单独的 `V_PAUSE` 状态名称。

**完成 (DONE)**

```
DONE VFA/ALK
PH0:6.89 V1:3.4
V2:3.6 VFA:11.0
ALK:6.0                ← 最终页明确使用 PH0 / V1 / V2 / VFA / ALK
```

### 8.2 状态转移图

以下为当前已确认的联合 VFA/ALK 实际状态逻辑；普通 pH 调节不是该测量的前置步骤：

```text
空闲(IDLE) --D4--> 联合VFA/ALK观察(RS_VFA) --合格--> S1脉冲滴定/混合复测到5.1 --S2脉冲滴定/混合复测到3.5 --计算--> 完成(RS_DONE)
```

普通 pH 调节与自动滴定会先把当前 `TT/TP/TL/TM/TD` 下发到 MCU，收到 ACK 后才真正 `START` 或进入自动补液，因此 OLED 的 `TGT` 会跟随最新确认目标。

```
                    D4按下
     ┌──────────────────────────────────────────┐
     │                                          │
     v                                          │
  RS_IDLE ──D4──> RS_TITR ──滴定──> RS_MIX     │
    ^                           ^      │        │
    │                           │      v        │
    │                         循环   达标        │
    │                           │      │        │
    │                           └──────┘        │
    │                              │            │
    │                   D4按下     v            │
    └── RS_DONE <────────────── RS_VFA         │
                    15秒后         │            │
                                  └─超时取消────┘
```

### 9.5 ORP 模组 Modbus 协议（内部）

ORP 模组使用 Modbus RTU 协议，Arduino 每 500ms 发送一次读取指令：

**请求帧 (8 字节)**：

```
FE 03 00 00 00 0B CRC_L CRC_H
│  │  └───┬───┘ └─┬─┘ └──┬──┘
地址 功能码 寄存器 寄存器 CRC16
           起始地址 数量
```

**响应帧 (25 字节)**，包含：

| 字节偏移 | 内容 |
|---------|------|
| 0 | 地址 0xFE |
| 1 | 功能码 0x03 |
| 4 | ORP 状态 (0=RUN, 1=CAL, 2=CALOK, 3=CALFAIL) |
| 5-6 | ORP 毫伏值 (带符号 16bit) |
| 7-8 | ORP 温度补偿毫伏值 (带符号 16bit) |
| 9-10 | 温度值 (×10, 例 253 = 25.3℃) |
| 14 | NTC 状态 (0=正常, 1=开路, 2=短路) |
| 21-22 | ORP AD 内码 |
| 23-24 | 温度 AD 内码 |

> 一般用户无需关心 Modbus 协议细节，只需知道 ORP 模组通过 D2/D3 与 Arduino 通信即可。

## 10. 标定与历史管理

### 10.1 标定原理

pH 值与 ORP 毫伏值之间存在近似线性关系：

```
pH = ORP_mV × K + B
```

其中 K 为斜率，B 为截距。通过测量 2 个或更多已知 pH 值的标准缓冲溶液在不同 pH 下的 ORP 读数，用最小二乘法计算出 K 和 B 系数。R² 值越接近 1，说明线性拟合越好。

### 10.2 标定操作步骤

1. 将 ORP 电极浸入 **已知 pH 值的标准缓冲液**（如 pH 4.01、pH 6.86、pH 9.18）。
2. 在 PC 上位机中，点击"标定"按钮，弹出标定窗口。
3. 选择标定点数（建议 2~3 点），为每个点输入**已知 pH 值**。
4. 依次点击每个点的"采集"按钮（或点击"全部采集"自动按序采集）：
   - 系统自动采集 30 帧 ORP 数据。
   - 自动去除最大 10% 和最小 10% 后取平均值。
   - 显示平均 ORP 毫伏值。
5. 所有点采集完成后，点击"计算"按钮。
6. 系统显示线性回归结果：K、B、R²。
7. 点击"应用"按钮，公式即刻生效并自动保存到 `calibrations.json`。
8. 应用或切换历史后，上位机会立即发送 `TK/TB` 并等待匹配 ACK；只有 ACK 成功才会标记“MCU 已应用”。

### 10.3 标定注意事项

- 电极在缓冲液中需稳定 1~2 分钟后再采集，确保读数稳定。
- 建议每次实验前重新标定。
- R² < 0.95 时建议检查电极状态或重做标定。
- 更换 ORP 电极后必须重新标定。

### 10.4 标定历史管理

- 每次标定结果自动保存到 `上位机/calibrations.json` 文件中，含时间戳、标签、K、B、R² 和原始数据点。
- 在"滴定控制"面板的"历史"下拉框中，可随时切换已保存的标定方案。
- 最多保留 50 条历史记录，超出后自动删除最旧的记录。
- 选择"— 默认 —"可恢复出厂公式 (K=1/200=0.005, B=4.0)。
- 当前 K/B 在低 pH 条件下若仅与试纸颜色大致落在同一量级，只能说明趋势基本一致；试纸分辨率不足以承担精密标定，绝对精度仍需准确标准缓冲液确认。

## 12. 故障排查

### 12.1 COM 口被占用

**现象**：连接时提示 "Access denied" 或 "Access is denied"。

**原因**：其他程序（Arduino IDE 串口监视器、其他串口工具）正在占用该 COM 口。

**解决方法**：
1. 关闭所有可能使用串口的程序（重点检查 Arduino IDE 的串口监视器）。
2. 如果找不到占用程序，可尝试重新插拔 Arduino USB 线。
3. 在 Windows 设备管理器中确认 COM 口号并尝试更换 USB 端口。

### 12.2 OLED 屏幕不显示或显示乱码

**现象**：OLED 黑屏、花屏、或显示内容不刷新。

**排查步骤**：
1. 检查 I2C 接线：SDA -> A4, SCL -> A5，VCC -> 5V，GND -> GND。
2. 用万用表测量 OLED 供电是否正常（VCC 与 GND 之间应为 5V ± 0.25V）。
3. 如果偶尔花屏：可能是供电不稳，尝试在 Arduino 5V 和 GND 之间并联 100μF 电解电容。
4. 如果完全无显示：检查 OLED I2C 地址是否为 0x3C（部分模块为 0x3D，若为 0x3D 需修改代码中 `OLED_ADDR` 宏定义）。
5. 固件使用裸 I2C 驱动，不依赖任何第三方 OLED 库，如果之前安装过 Adafruit 库也不会影响运行。

### 12.3 泵不响应

**现象**：发送开泵指令但泵不转，或泵一直转不停。

**排查步骤**：
1. 检查继电器模组的触发方式（高电平触发 vs 低电平触发）。
   - 固件以 `HIGH` 为开泵，`LOW` 为关泵。
   - 如果继电器模组为低电平触发，需要修改代码或在外部加装反相电路。
2. 测量 D11/D12/D13 引脚电压：
   - 发送开泵指令后，对应引脚应为 5V (HIGH)。
   - 发送关泵指令后，对应引脚应为 0V (LOW)。
3. 检查继电器模组供电（通常需要 5V 或 12V 独立电源）。
4. 检查蠕动泵电源是否接通，泵管安装是否到位。

### 12.4 上位机无数据

**现象**：连接成功但数据面板不更新，pH 曲线空白。

**排查步骤**：
1. 确认 Arduino 已烧录正确的固件。
2. 用 Arduino IDE 串口监视器（波特率 115200）查看是否有 `ORP PH V10 START` 以及后续 `STS:` / `ORPMV:` 等数据输出。若没有，说明固件未正确运行。
3. 若串口监视器有数据但上位机不显示，检查上位机调试日志区域是否显示 `← STS:RUN` 等接收数据，确认 COM 口和波特率是否正确。
4. 确认 ORP 模组接线正确（TX->D2, RX->D3），ORP 模组供电正常。

### 12.5 pH 读数异常

**现象**：pH 值显示异常、主界面 pH 与 MCU 诊断 pH 不一致，或 OLED 与上位机目标不一致。

**排查步骤**：
1. 检查是否已完成标定，标定后的 K、B 系数是否合理。
2. 在调试日志中核对 `ORPMV:` 行的原始毫伏值是否正常（正常范围通常在 ±2000mV 以内），并确认主界面 pH 使用 `ORPMV × K + B`。
3. 若 OLED/MCU pH 与主界面不一致，检查最近一次 `TK/TB` 是否已经收到匹配 ACK；未确认时，上位机会阻止 `START`、自动滴定和 `VFA`。
4. 检查 ORP 电极是否已连接到 ORP 模组，电极是否损坏或干涸。
5. 如果 ORP 状态显示 `CALFAIL`，说明 ORP 模组内部校准失败，需检查电极或更换 ORP 模组。

### 12.6 脱机流程异常

**现象**：按 D4 无反应，或流程卡在某阶段不动。

**排查步骤**：
1. 确认 OLED 显示 `READY. PRESS BTN` 状态（只有在 IDLE 状态按 D4 才有效）。
2. 如果在 DONE 状态，按 D4 应复位到 IDLE，再按一次启动。
3. 检查 D4 按键接线：一端接 D4，另一端接 GND。按下时 D4 应读到 LOW。
4. VFA 联合测量改为“脉冲滴定 + 混合复测”后，正式滴定最长允许 1000 秒；若到时仍未到终点，会因 `TIMEOUT` 自动安全停泵并返回空闲。可检查酸浓度 (`FN`)、样品体积 (`FS`)、电极响应和泵流速标定。

### 12.6.1 VFA:REJECT LOW_PH / UNSTABLE

**现象**：上位机提示 `LOW_PH` 或 `UNSTABLE`，且本次没有产生新结果。

**处理规则**：
1. `LOW_PH` 表示 10 秒平均起始 pH `< 5.5`。不要降低阈值，也不要自动加碱补救。
2. 如果样品已经被普通 pH 调节到 5.0 左右，这一份样品不能继续用于联合 VFA/ALK；必须重新取符合条件的原始样品。
3. `UNSTABLE` 表示 10 秒极差 `> 0.10`。请等待样品稳定、检查搅拌/电极状态，必要时重新取样。
4. `普通调节目标 pH` 仅用于普通 pH 调节流程，与联合 VFA/ALK 的起始准入无关。

### 12.6.2 VFA:CALC_ERROR

**现象**：端点已经完成，但上位机提示 `VFA:CALC_ERROR ...`，本次没有新增有效结果。

**处理规则**：
1. 这类错误通常表示分母、有限值或物理合理性检查未通过，系统会把本次结果标记为无效，而不是静默保存为 `0/0`。
2. 上位机会保留上一条有效结果，不会用这次失败覆盖它。
3. 请结合同次上报的 `VFA:TRACE PH0/A1/A2/TOTAL/FN/FS/BLANK` 复盘端点、酸浓度和样品体积是否合理。
4. 当前版本中的 `0.25 mL` 为空白滴定修正；若现场流程不同，应在后续版本中改为可配置值，而不是手工改公式符号。

### 12.7 参数无法保存

**现象**：断电后设置的参数丢失，恢复到默认值。

**排查步骤**：
1. Arduino 的 EEPROM 写入寿命约为 10 万次。如果程序频繁写入 EEPROM，可能已耗尽寿命。
2. 正常使用时，参数仅在 PC 发送设置指令或泵停止时写入，寿命应足够长时间使用。
3. 如果确认 EEPROM 损坏，唯一解决方法是更换 Arduino UNO 的 ATmega328P 芯片。

### 12.8 Python 启动错误

**常见错误**：
- `No module named 'serial'` -> 执行 `pip install pyserial`
- `No module named 'matplotlib'` -> 执行 `pip install matplotlib`
- `ModuleNotFoundError: No module named 'tkinter'` -> Python 安装时未勾选 tkinter，重新安装 Python 并确保勾选或选择 "完整安装"

## 13. 泵流量标定

### 13.1 功能说明

泵流量标定用于重新测得三路泵的真实流速，并将结果保存为可追溯的历史记录。第一版采用量筒体积法，支持碱泵、酸泵、水泵分别独立标定。

核心规则如下：

- 正式标定运行由 MCU 负责计时和自动停泵，上位机只负责发送配置、接收实际运行时间、录入人工量筒体积并计算流速。
- 预充液和正式标定排出的液体都不计入业务累计体积。
- 标定会话期间，普通手动泵控制、PC 自动滴定、VFA 测量与脱机自动流程都不得同时启动。
- D4 在标定运行或预充液期间变为紧急停止键；空闲时仍保持原有 VFA 启动语义。
- 老固件若不支持 FCAL 协议，上位机会禁用预充与运行按钮，不允许退化为 PC 本地计时。

### 13.2 入口位置

在上位机右侧“泵流速 (ml/s)”区域中：

- 三个流速输入框保留，用于手动输入。
- 每行新增“应用”按钮，只有确认后才会把流速下发到 MCU。
- 区域底部新增“流量标定”按钮，点击后打开独立中文窗口。

### 13.3 手动流速应用

手动应用流程：

1. 在碱泵、酸泵或水泵对应输入框中输入新流速。
2. 按 Enter 或点击该行“应用”按钮。
3. 确认对话框会显示泵名和 6 位小数流速。
4. 上位机向 MCU 发送 `FB/FA/FW <value>`。
5. 只有收到对应 ACK 后，新的流速、来源状态和当前生效记录才会更新。

来源显示规则：

- `来源: 默认`
- `来源: 手动`
- `来源: 标定 <记录ID>`

### 13.4 泵流量标定窗口

窗口分为四块：

1. 基本信息：泵、标定时长、液体名称、管规格、泵/通道、温度、备注。
2. 运行状态：固件 FCAL 能力、当前状态、预充/标定/停止按钮。
3. 待回填运行：显示 MCU 实际运行结果，并录入量筒体积。
4. 历史记录：显示当前泵的历史标定记录，支持应用、删除、CSV 导出。

### 13.5 预充液

预充液用于排气或润管，不计入测量点：

1. 选择泵和液体条件。
2. 点击“开始预充”。
3. 如需提前停止，点击“立即停止”或按 D4。
4. MCU 还会在 30 秒时自动硬停。
5. 预充结束后不会生成待回填点位，只在状态区提示结束原因。

### 13.6 正式标定运行

正式标定步骤：

1. 选择待标定的泵。
2. 选择时长。默认 10 秒，可快捷选择 5/10/30/60 秒，也可手填 3–120 秒。
3. 点击“开始标定”。
4. 弹出安全确认，明确提示：
   - 泵名
   - 时长
   - 液体名称
   - 出口必须放入量筒
   - 本次标定液不计入累计体积
5. 确认后 MCU 启动泵、独占计时，并在到时后自动停泵。
6. 若测试人员点击“立即停止”，且 MCU 实际运行时间不少于 3 秒，则该点位可作为有效提前停止点；若不足 3 秒则作废。
7. 若因 D4 紧急停止、错误或断连导致中止，则该次运行无效，不可保存为点位。

### 13.7 待回填与点位保存

正式运行完成后，会进入“待回填运行”状态：

1. 上位机显示泵、事件、计划时间、MCU 实际时间和结束原因。
2. 在“量筒体积 (ml)”中输入实测体积。
3. 体积范围为 `0.01–5000.00 ml`，最多保留 2 位小数。
4. 点击“保存为下一点”后，会生成一个候选点位。
5. 候选点位会在窗口中显示为 `1/3`、`2/3`、`3/3` 进度。
6. 候选点位已经保存后，不再属于“待回填运行”；同一标定会话、同一泵和同一泵送配置下，允许继续开始第 2 / 3 个正式标定点。
7. 候选点位存在期间，泵选择、液体/管规格/泵送配置等会继续锁定，普通泵控制、普通 pH 调节、VFA/ALK 和 PC 自动滴定仍保持全局互斥。
8. 达到 `3/3` 后，不再允许继续正式标定；测试人员必须选择“保存记录”、“保存并应用”或“清空本次点位”。

每个点位至少包含：

- 计划时间
- MCU 实际时间
- 人工量筒体积
- 单点流速
- 是否为有效提前停止点
- 结束原因

注意：

- 每次记录最多 3 个点位。
- 允许只用 1 点或 2 点保存并应用。
- 同一组 1 / 2 / 3 个点允许使用不同的标定时长。
- 存在待回填运行时，不允许再次开始预充或正式标定。
- 已经保存为候选点的点位不会阻止同一会话继续采集下一点，但会继续阻止其他业务流程插入。

### 13.8 流速计算与告警

最终流速使用加权公式：

`最终流速 = 总体积 / MCU 总实际时间`

显示与保存规则：

- 窗口内汇总显示 4 位小数。
- 下发 MCU 和配置持久化时保存 6 位小数。
- 若单点流速相对各点流速的算术平均值偏离超过 10%，窗口会显示“偏离 xx.x%”警告，但不阻止保存。
- 若最终流速超出 `0.001–100 ml/s`，保存前会要求二次确认。

### 13.9 保存、应用与历史

点击“保存记录”：

- 仅保存到历史，不修改 MCU 当前流速。

点击“保存并应用”：

1. 先保存记录。
2. 再发送 `FB/FA/FW` 到 MCU。
3. 只有收到对应 ACK 才会：
   - 更新上位机当前流速数值
   - 将来源改为 `标定`
   - 将该历史记录标记为当前生效
4. 如果收到 ERR 或 BUSY：
   - 历史记录仍然保留
   - 当前生效流速与来源保持不变
5. 如果等待 ACK 超时、ACK 数值与请求值不一致，或串口中途断开：
   - 历史记录仍可保留
   - 当前生效流速、来源和输入框都会回退到旧值
   - 不会把失败草稿写入持久化配置

历史规则：

- 每台泵最多保留 50 条记录。
- 当前生效记录不能直接删除，必须先切换到其他记录或改为手动流速。
- 删除仅影响历史，不会改写已经导出的 CSV。
- CSV 导出包含摘要字段和每个点位的明细行。

### 13.10 标定与其他流程互斥

开始条件必须全部满足：

- MCU 当前空闲
- 三路泵全部停止
- PC 自动滴定已关闭
- 不存在待回填运行

互斥失败时，MCU 会返回 `ERR:FCAL BUSY`，上位机会在状态栏与说明区显示原始串口信息。

### 13.11 串口协议摘要

上位机使用的泵流量标定协议如下：

- 探测能力：`FCAL?`
- 查询状态：`FCAL STATUS`
- 预充液：`FCAL PRIME B|A|W`
- 正式运行：`FCAL RUN B|A|W <seconds>`
- 停止：`FCAL STOP`

常见上报：

- `FCAL:CAPS V1 PRIME RUN STOP STATUS`
- `FCAL:STATE RUN PUMP:B PLAN_MS:10000 ELAPSED_MS:4000`
- `FCAL:DONE PUMP:B MODE:RUN PLAN_MS:10000 ACTUAL_MS:10000 REASON:AUTO`
- `FCAL:STOPPED PUMP:B MODE:RUN PLAN_MS:10000 ACTUAL_MS:6500 REASON:USER`
- `FCAL:ABORTED PUMP:B MODE:RUN PLAN_MS:10000 ACTUAL_MS:1200 REASON:BUTTON`

### 13.12 现场注意事项

- 每次正式运行前，必须确认出口已插入量筒，且不会喷溅到设备。
- 预充液只用于排气，不可当作测量点。
- 必须佩戴护目镜、耐化学品手套、实验服。
- 软管出口必须固定，量筒或接液容器必须放在接液盘内，避免喷溅和溢漏。
- 酸、碱、水应分别使用清洁容器，避免交叉污染。
- 标定液和废液必须按实验室规定处置，禁止直接回倒样品或随意排放。
- 若关闭标定窗口、断开串口或退出程序时仍在运行，上位机会先发送 `FCAL STOP`，只有收到 MCU 终态后才允许关闭/断开。
- 如果等待终态确认超时，上位机会取消退出/断开并保留连接，必须现场确认泵已停、必要时再次点“立即停止”。
- 若程序断连或重启，未保存的待回填数据会作废，不会跨重启恢复。
- 泵流速更新后，不会回算历史累计体积、历史 VFA 结果或既有导出文件。
